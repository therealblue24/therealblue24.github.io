Byte pair encoding: Underestimated, Underrated

Byte pair encoding is a compression scheme where parts of the input are mapped to a character that is not in the input.

Example of byte-pair encoding

Suppose we want to compress the message aaabdaaabac.

We can see that the character pair aa occurs most often. So we can replace it with the byte (or character) Z, because it's not used in the message.

We now have ZabdZabac, where Z is aa.

Then the process is repeated with byte pair ab, replacing it with Y.

We now have ZYdZYac, where Z is aa and Y is ab.

We see that ZY is used 2 times, so we can just replace it with X.

We now have XdXac, where Z is aa and Y is ab and X is ZY.

We cannot continue this process further because there is no more logical byte pairs we can choose.
